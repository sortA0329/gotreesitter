# Linguist Language Name Mapping

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Let callers pass any linguist/enry language name (e.g., "C++", "Golang", "Shell") into gotreesitter and get the right grammar back — and retrieve display names without maintaining their own mapping tables.

**Architecture:** A code generator fetches GitHub Linguist's `languages.yml`, matches each gotreesitter grammar to its linguist entry, and emits a generated Go file with alias→grammar and grammar→display-name maps. Runtime lookup is a single `DetectLanguageByName()` call.

**Tech Stack:** Go, YAML parsing (gopkg.in/yaml.v3), no runtime dependency on go-enry.

---

## The Problem

Callers like Gitea maintain 4 layers of glue to bridge enry/linguist names to gotreesitter:

1. `normalizeFileNameLang()` — enry quirk fixes (F# → FSharp, C with .C → C++)
2. `canonicalLanguageKey()` — character transforms (+ → p, # → sharp)
3. `treeSitterLanguageAliases` — 12 explicit aliases (golang → go, shell → bash)
4. `treeSitterDisplayName()` — 15 hardcoded reverse mappings (cpp → "C++")

Every new language or enry naming change requires a caller-side update. This mapping belongs in gotreesitter.

---

## Data Flow

```
linguist/languages.yml (GitHub)  +  languages.manifest (local)
    ↓
cmd/ts2go gen-linguist (generator)
    ↓
grammars/linguist_gen.go (generated)
    ↓
grammars/registry.go  DetectLanguageByName() / DisplayName()
```

## Generator Design

### Input: Linguist's languages.yml

Fetched from `https://raw.githubusercontent.com/github-linguist/linguist/master/lib/linguist/languages.yml` at generation time.

Each entry provides:
```yaml
Go:
  type: programming
  color: "#00ADD8"
  aliases:
  - golang
  extensions:
  - ".go"
  tm_scope: source.go
  ace_mode: golang
  language_id: 132
```

We extract: canonical name, aliases, type.

### Matching: Linguist ↔ gotreesitter

For each grammar in `languages.manifest`, the generator finds its linguist entry by:

1. **Normalize and compare**: lowercase the linguist canonical name, strip spaces/special chars, compare to grammar name. Handles ~190 of 206 grammars automatically.
2. **Check aliases**: if the grammar name appears in a linguist entry's aliases list (e.g., "cpp" in C++'s aliases).
3. **Explicit overrides**: a hardcoded map in the generator for the ~15 cases that can't be auto-matched:

```go
var linguistOverrides = map[string]string{
    // gotreesitter name → linguist canonical name
    "c_sharp":    "C#",
    "cpp":        "C++",
    "objc":       "Objective-C",
    "fsharp":     "F#",
    "make":       "Makefile",
    "tsx":        "TSX",
    "commonlisp": "Common Lisp",
    "bash":       "Shell",
    // ... ~15 total, discovered during generation
}
```

Grammars with no linguist match (e.g., `markdown_inline`, `regex`, `comment`) get no aliases and a default title-cased display name.

## Runtime API

Added to `grammars/registry.go`:

```go
// DetectLanguageByName returns the LangEntry for any linguist name,
// alias, or gotreesitter grammar name. Returns nil if unknown.
//
// Accepts: "C++", "cpp", "Go", "golang", "Shell", "bash", etc.
func DetectLanguageByName(name string) *LangEntry {
    key := normalizeLinguistKey(name)
    if grammarName, ok := linguistToGrammar[key]; ok {
        return lookupByName(grammarName)
    }
    // Fall back to direct registry name match.
    return lookupByName(key)
}

// DisplayName returns the canonical display name for a language.
// Returns the linguist name if known (e.g., "C++" for cpp),
// otherwise title-cases the grammar name.
func DisplayName(entry *LangEntry) string {
    if entry == nil {
        return ""
    }
    if name, ok := grammarDisplayNames[entry.Name]; ok {
        return name
    }
    return strings.Title(strings.ReplaceAll(entry.Name, "_", " "))
}

// normalizeLinguistKey lowercases and trims the input but preserves
// special characters like +, #, / so "C++" maps correctly.
func normalizeLinguistKey(name string) string {
    return strings.TrimSpace(strings.ToLower(name))
}
```

Key difference from Gitea's `canonicalLanguageKey`: we do NOT strip `+`, `#`, etc. Instead, the generated map contains keys with those characters preserved (lowercased). This eliminates the lossy normalization that caused Gitea to need special-case handling.

## What Gitea Can Delete After This

Once gotreesitter exposes `DetectLanguageByName` and `DisplayName`:

- `canonicalLanguageKey()` — replaced by `DetectLanguageByName`
- `treeSitterLanguageAliases` map — absorbed into generated `linguistToGrammar`
- `treeSitterDisplayName()` — replaced by `DisplayName`
- `normalizeFileNameLang()` — the F#/Pascal/C cases are handled by the alias map
- The entire `treeSitterRegistry().byCanonicalName` index — replaced by gotreesitter's own lookup

## Validation

- Every grammar in `languages.manifest` should appear in `grammarDisplayNames` (100% coverage)
- Every linguist entry that matches a grammar should have all its aliases in `linguistToGrammar`
- Generator reports unmatched grammars (no linguist entry) as warnings
- Generator reports ambiguous matches (multiple linguist entries for one grammar) as errors
- Unit tests: `DetectLanguageByName("C++")` returns cpp, `DisplayName` for cpp returns "C++", etc.
- Integration: run Gitea's highlight tests with the new API to confirm the mapping layer can be removed
